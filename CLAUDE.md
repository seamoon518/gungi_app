- 内容について質問や疑問があれば積極的にすること。
- 私が書いていることが絶対的に正しいわけではなく、目的を達成するためのより簡単で最適なアプローチがあれば、実装前に提案すること。

# 実行の承認・確認プロセスについて（重要）
- 開発スピードを最大化するため、ファイルの作成・編集、テストの実行、パッケージのインストールなど、通常の開発操作においては事前確認は不要です。事後報告のみで自律的に作業を進めてください。
powershell の動作も許可不要
- ただし、以下の「破壊的な変更」や「影響範囲が大きい操作」を行う場合のみ、必ず実行前に確認を求めてください：
  - フォルダやファイルの削除
  - 大規模なリファクタリングや、既存のコアロジックを大きく書き換える場合
  - データベースのスキーマ変更やデータ削除
  - 本番環境へのデプロイ操作

# アーキテクチャ
- フロントエンド: Next.js (App Router), TypeScript
- バックエンド: Python, FastAPI

# セキュリティとコーディング規約
- デプロイを前提としているため、APIキーやデータベースのパスワードなどの機密情報は、絶対にソースコード内に直接記述（ハードコード）しないこと。
- 必ず `.gitignore` や `.env` を使用して管理すること。

# 外部APIの利用ルール
- 外部API（Gemini APIなど）を呼び出す処理は、セキュリティの観点からフロントエンド（Next.js側）には直接記述しないこと。
- 外部通信やAPIキーを必要とする処理は、必ずバックエンド（FastAPI側）にエンドポイントを作成し、そこを経由して実行すること。

# よく使うコマンド
- フロントエンド起動: `npm run dev`
- バックエンド起動: `uvicorn main:app --reload`
